# STM32F103C8T6 智能环境监测终端

基于 STM32F103C8T6（Blue Pill）的联网版智能环境监测系统，集成温湿度采集、超声波测距、人体红外感应、电机控制、Flash 数据持久化、OLED 显示、蜂鸣器报警、编码器交互以及 ESP8266 WiFi 局域网 Web 服务器等功能。可作为智能家居环境监测节点或嵌入式学习参考项目。

---

## 功能概览

### 1. 环境监测

系统通过三个传感器实时采集环境数据：

- **DHT11 温湿度传感器**：每 2 秒读取一次温度和湿度，精度为整数位 + 1 位小数
- **HC-SR04 超声波测距**：每 200ms 测量一次前方障碍物距离，量程 2~400 cm
- **HC-SR501 PIR 人体红外感应**：中断触发实时检测，响应延迟 < 1 秒

### 2. 自动控制与报警

系统根据实时数据自动执行控制逻辑：

| 触发条件 | 响应动作 |
|---------|---------|
| 温度 > 阈值 | 自动启动散热风扇，转速 = (温度 - 阈值) × 10%，上限 100% |
| 湿度 > 阈值 | 蜂鸣器慢速报警（1 秒周期，响 100ms） |
| 距离 < 阈值 | 蜂鸣器急促报警（200ms 周期，响 100ms） |
| PIR 检测到人体 | 黄色 LED 亮 1 秒，蜂鸣器短响 50ms 提示 |

**报警优先级**：距离报警 > 湿度报警 > PIR 提示。高优先级报警会覆盖低优先级。

### 3. OLED 仪表盘

128×64 像素 OLED 分 4 行实时显示：

```
T:25.5C H:65.0%      ← 温湿度
Dist:45cm            ← 超声波距离
Motion:YES           ← PIR 人体检测状态
Fan:50% Alm:A        ← 风扇转速 / 报警状态
```

- `Alm:A` 表示有报警激活（距离/湿度/温度任一异常）
- `Alm:N` 表示无报警
- 距离读取失败时显示 `Dist:ERR`

### 4. 菜单系统

通过 **EC11 旋转编码器** 操作菜单：

**菜单项列表**：
1. **Temp Threshold** — 温度报警阈值（20~50°C，默认 30°C）
2. **Hum Threshold** — 湿度报警阈值（50~100%，默认 80%）
3. **Dist Threshold** — 距离报警阈值（5~100 cm，默认 10 cm）
4. **Rec Interval** — 历史记录保存间隔（1/5/10 分钟，默认 5 分钟）
5. **Fan Manual** — 手动风扇控制（0~100%，设为 0 关闭手动模式）
6. **History** — 浏览最近 20 条历史记录
7. **Clear History** — 清空所有历史记录（带确认对话框）
8. **Back** — 返回仪表盘

**交互方式**：

| 操作 | 仪表盘界面 | 菜单界面 | 编辑/浏览界面 |
|------|-----------|---------|--------------|
| 编码器 **短按** | 进入菜单 | 确认选中项 | 保存并返回 |
| 编码器 **旋转** | 无作用 | 上下切换菜单 | 调整数值 / 翻页 |
| KEY_BACK **短按** | 关闭手动风扇 | 返回仪表盘 | 返回上级（取消） |

> **快捷操作**：Dashboard 界面下按 KEY_BACK 可直接关闭手动风扇，无需进入菜单。

### 5. 历史记录与掉电保护

- **RAM 缓存**：最近 20 条环境数据保存在 RAM 中，支持 OLED 翻页浏览
- **Flash 持久化**：所有历史记录同步写入 W25Q64 Flash（Sector 1），最多存储 511 条，**掉电不丢失**
- **自动覆盖**：Flash 写满后自动擦除 Sector，从头开始覆盖最早记录
- **开机加载**：系统启动时自动从 Flash 读取最近 20 条记录到 RAM
- **手动清空**：菜单中提供 "Clear History" 选项，确认后清空 Flash 和 RAM 中的所有记录

每条记录包含：时间戳（开机秒数）、温度（×10）、湿度（×10）。

### 6. 参数掉电保存

温度/湿度/距离阈值和记录间隔等参数在修改后自动保存到 W25Q64 Flash（Sector 0），重启后自动恢复。首次上电时若 Flash 中无有效参数，系统自动写入默认值。

### 7. WiFi Web 服务器

ESP8266 模块通过 AT 指令连接 WiFi 热点，开启 TCP Server（端口 80）。手机/电脑连接同一热点后，浏览器访问 ESP8266 的 IP 地址即可查看实时环境数据网页。

网页显示内容：温度、湿度、距离、人体检测状态、风扇转速，带响应式 CSS 样式。

### 8. LED 状态指示

| LED | 颜色 | 状态 |
|-----|------|------|
| 绿色 | 常亮 | 系统正常运行 |
| 黄色 | 亮 1 秒 | PIR 检测到人体 |
| 红色 | 500ms 闪烁 | 有报警激活（温度/湿度/距离异常） |

### 9. 低功耗显示

10 秒无操作（无按键、无旋转）后 OLED 自动熄屏进入低功耗状态。任意编码器旋转或按键操作立即唤醒并恢复显示。

## 构建与烧录

### 环境要求

- ARM GCC 工具链（`arm-none-eabi-gcc`，推荐 STM 官方捆绑版）
- CMake >= 3.22
- Ninja
- STM32CubeMX（可选，用于修改时钟、引脚或外设配置后重新生成代码）

### 构建命令

```bash
# 配置 Debug 构建
cmake --preset Debug

# 编译
cmake --build --preset Debug
```

构建输出为 `build/Debug/Jack.elf`。Release 构建将 `--preset Debug` 替换为 `--preset Release` 即可。

### 烧录

**方式一：VS Code + STM32CubeIDE 扩展**

项目已配置 `.vscode/launch.json`，点击 VS Code 侧边栏的"运行和调试"即可通过 ST-Link 烧录并启动 GDB 调试。

**方式二：命令行（OpenOCD）**

```bash
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg \
    -c "program build/Debug/Jack.elf verify reset exit"
```

## 使用指南

### 首次上电

1. 接通电源后绿色 LED 常亮，OLED 显示仪表盘
2. ESP8266 自动尝试连接配置的 WiFi 热点（默认 `ningkai`）
3. 若 Flash 中无有效参数，系统自动写入默认阈值

### 设置报警阈值

1. 编码器短按进入菜单
2. 旋转选中 "Temp Threshold" / "Hum Threshold" / "Dist Threshold"
3. 短按进入编辑，旋转调整数值
4. 短按保存（自动写入 Flash），或按 KEY_BACK 取消

### 手动控制风扇

1. 菜单中选择 "Fan Manual"
2. 旋转调整转速（0~100%）
3. 短按保存
4. **Dashboard 界面下按 KEY_BACK 可一键关闭**

### 查看历史记录

1. 菜单中选择 "History"
2. 旋转翻页查看最近 20 条记录
3. 短按或 KEY_BACK 返回菜单

### 清空历史记录

1. 菜单中选择 "Clear History"
2. 确认界面显示 "Clear History?"
3. 短按确认清空，KEY_BACK 取消

### 查看 Web 页面

1. 确保 ESP8266 已成功连接 WiFi（手机/电脑连接同一热点）
2. 查看 ESP8266 获取的 IP 地址（可通过串口调试助手查看 AT 指令返回）
3. 浏览器访问 `http://<ESP8266_IP>`

## WiFi 配置

在 `Core/MY/esp8266.h` 中修改热点信息后重新编译：

```c
#define ESP8266_SSID     "你的热点名称"
#define ESP8266_PASS     "你的热点密码"
```

ESP8266 上电初始化流程（自动执行，无需手动干预）：

```
AT+RST          → 模块复位
AT+CWMODE=1     → 设为 Station 模式
AT+CWJAP        → 连接 WiFi 热点
AT+CIPMUX=1     → 开启多连接模式
AT+CIPSERVER=1,80 → 开启 TCP Server（端口 80）
```

## 核心技术实现

### HC-SR04 超声波测距

采用 **EXTI3 双边沿中断 + TIM4 自由运行计数器** 方案：

- TRIG 引脚发送 10μs 高电平触发
- ECHO 引脚上升沿触发 EXTI，记录 TIM4 计数值
- ECHO 引脚下降沿再次触发 EXTI，计算时间差
- 距离 = 时间差(μs) / 58，量程 2~400 cm

> 相比纯轮询方案，中断方案不受主循环阻塞影响，测距更稳定。

### PIR 与 HC-SR04 EXTI 共享

两者均使用 `HAL_GPIO_EXTI_Callback`，在 `pir.c` 中通过 `GPIO_Pin` 判断来源：

- `GPIO_Pin == PIR_PIN`（PB5）→ PIR 人体检测处理
- `GPIO_Pin == HCSR04_ECHO_PIN`（PA3）→ HC-SR04 回响处理

### DHT11 单总线时序

使用 **DWT（Data Watchpoint and Trace）周期计数器**实现 μs 级精确延时：

- 发送起始信号（拉低 18ms + 释放 30μs）
- 读取 40 位数据（每位 50μs 低电平 + 26~70μs 高电平）
- 两次读取间隔必须 ≥ 1 秒

### 编码器 EMI 滤波

电机运行时电磁干扰可能导致编码器误触发。软件层面采用三层滤波：

1. **20ms 采样限频**：编码器只每 20ms 读取一次
2. **阈值过滤**：原始变化量 ≤ ±2 视为抖动，直接丢弃
3. **连续方向确认**：连续 2 次同方向采样（40ms）才确认有效转动

### ESP8266 通信

USART1 中断接收 + AT 指令状态机：

- `HAL_UART_RxCpltCallback` 单字节中断接收，存入环形缓冲
- 主循环中 `ESP8266_Task()` 解析响应，管理 WiFi 连接和 HTTP 请求
- 收到 `GET /` 请求后组装 HTML 页面并通过 `AT+CIPSEND` 发送

## 故障排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| OLED 不显示 | I2C 接线错误 / 地址不匹配 | 检查 PB6/PB7 接线，确认 OLED 地址为 0x78 |
| DHT11 读取失败 | 两次读取间隔太短 / 接线松动 | 确保间隔 ≥ 1 秒，检查 PA8 上拉 |
| HC-SR04 始终 ERR | ECHO 未触发 / 超时 | 检查 PA2/PA3 接线，确认 TIM4 已启动 |
| PIR 不触发 | 灵敏度太低 /  warm-up 时间 | PIR 上电后需 10~60 秒稳定期 |
| 编码器乱跳 | 电机 EMI 干扰 | 已软件滤波，如仍严重可给编码器信号线加 100nF 电容 |
| ESP8266 连不上 WiFi | 密码错误 / 信号弱 / 供电不足 | 确认 SSID/密码正确，ESP8266 必须 3.3V 供电 |
| 蜂鸣器长鸣不停 | 距离/湿度持续异常 | 调整阈值或移开障碍物 |
