# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

STM32F103C8T6 (Blue Pill) firmware project using STM32 HAL library, built with CMake and the arm-none-eabi-gcc toolchain.

## Build Commands

This project uses CMake presets defined in `CMakePresets.json`:

```bash
# Configure Debug build
cmake --preset Debug

# Build Debug
cmake --build --preset Debug

# Configure and build Release
cmake --preset Release
cmake --build --preset Release
```

The build output is `build/Debug/Jack.elf` (or `build/Release/Jack.elf`).

### Direct CMake (without presets)

```bash
cmake -B build/Debug -DCMAKE_BUILD_TYPE=Debug -DCMAKE_TOOLCHAIN_FILE=cmake/gcc-arm-none-eabi.cmake -G Ninja
cmake --build build/Debug
```

### Flash / Debug

The VS Code `launch.json` is configured for ST-Link GDB Server. Use the STM32CubeIDE VS Code extension or OpenOCD to flash the target.

## Architecture

### MCU Configuration (CubeMX)

- **MCU**: STM32F103C8T6, LQFP48, 72 MHz max
- **Clock**: HSE 8 MHz crystal → PLL ×9 → 72 MHz SYSCLK, APB1 = 36 MHz, APB2 = 72 MHz
- **Peripherals configured**:
  - `TIM3`: PWM Generation Channel 3 on `PB0`
  - `GPIO`: `PB1` (labeled AIN2), `PB2` (labeled AIN1) as push-pull outputs
  - `SYS`: SWD debug (`PA13`/`PA14`), SysTick

### Project Structure

- **`Core/Src/` / `Core/Inc/`**: CubeMX-generated application code. User code must live inside `/* USER CODE BEGIN/END */` blocks or it will be lost on regeneration.
- **`Core/MY/`**: Custom user code directory (not managed by CubeMX). Add sources here and register them in the top-level `CMakeLists.txt`.
- **`cmake/stm32cubemx/CMakeLists.txt`**: **Auto-generated by CubeMX**. Do not edit directly—it will be overwritten. Add custom sources, includes, and definitions to the top-level `CMakeLists.txt` instead via `target_sources`, `target_include_directories`, etc.
- **`Drivers/`**: STM32 HAL driver and CMSIS.
- **`Jack.ioc`**: STM32CubeMX configuration file. Open in CubeMX to modify clocks, pins, or peripherals, then regenerate code.
- **`cmake/gcc-arm-none-eabi.cmake`**: GCC ARM toolchain configuration.
- **`.vscode/`**: VS Code settings for STM32CubeIDE extension (`cube-cmake`, `starm-clangd`).

### Adding Custom Code

1. **In CubeMX-generated files**: Place code only between `/* USER CODE BEGIN */` and `/* USER CODE END */` markers.
2. **New files**: Create them under `Core/MY/` (or another directory), then add them to the top-level `CMakeLists.txt`:
   ```cmake
   target_sources(${CMAKE_PROJECT_NAME} PRIVATE
       Core/MY/my_driver.c
   )
   target_include_directories(${CMAKE_PROJECT_NAME} PRIVATE
       Core/MY
   )
   ```

### HAL Modules Enabled

`HAL_GPIO_MODULE_ENABLED`, `HAL_TIM_MODULE_ENABLED`, plus the standard core modules (`RCC`, `CORTEX`, `DMA`, `FLASH`, `EXTI`, `PWR`).
